This is a beginner's guide to building accessible web application. More specifically, this is guide a to building websites in a way that individuals with disabilities (such as visual, physical or neurological) are be able to understand, interact and contribute to your website just as easily as a normal person can. If you are a web developer looking to create accessible websites and dont know where to start, this is the right place.

Tools that help individuals with disabilities are broadly termed as Assistive Technology (AT). ATs that help disabled users browse the web include:

  • Wands and sticks worn on the head, held in the mouth or strapped to the chin and used to press keys on the keyboard
  • Screen readers are used to verbalize, or "speak," everything on the screen including text, graphics, control buttons, and menus into a computerized voice that is spoken aloud. ChromeVox is one such (free) plugin for Google Chrome browser, other popular screen readers include JAWS, NVDA etc
  • Screen magnifiers work like a magnifying glass for the computer by enlarging a portion of the screen which can increase legibility and make it easier to see items on the computer

Much like how W3C defined HTML5 markup, Web Content Accessibility Guidelines (WCAG) defines guidelines to make web applications accessible. Which means that developers need to follow these guidelines in order to make their websites compatible with ATs. If you design your website in a way that will help screen reader do its job then you can empower a visually impaired user do exactly as much as a normal user can.
